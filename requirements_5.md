## Требования на оценку 5 (исключая требования на 3 и 4)

### Общие требования к проекту:

1. Реализовать **пользовательский интерфейс** (десктопное приложение).
2. В качестве интерфейса может быть использован: **web, desktop, bot (vk)**.  
   → В данном случае выбран **desktop** с использованием `tkinter`.
3. **Все разработанные точки входа (REST API из оценки 4) должны быть использованы** в клиентском приложении.

### Состав папки с вариантом (на оценку 5):

В папке `Sn` (где `n` – номер варианта) должны быть следующие файлы:

- `doc.md` – описание API и ER-диаграмма (из оценки 3, но требуется)
- `models.py` – модели БД (из оценки 3)
- `requirements.txt` – зависимости
- `service.py` – реализация REST API на FastAPI (из оценки 4)
- **`client.py` – всё клиентское приложение, включая логику**

### Требования к `client.py` (на `tkinter`):

- **Вся логика клиентского приложения должна быть внутри `client.py`**.
- Приложение должно:
  - Подключаться к REST API (FastAPI), реализованному в `service.py`.
  - Вызывать все методы API:
    - создание сущности
    - изменение сущности по ID
    - удаление сущности по ID (скрытое удаление, `is_active = False`)
    - получение сущности по ID
    - получение списка сущностей по заданным параметрам
  - Отображать результаты операций в удобном для пользователя виде (например, в таблицах, текстовых полях, логах).
- Интерфейс должен быть интуитивно понятным, с пояснениями полей ввода.
- Должна быть реализована **базовая обработка ошибок** (например, если сервис не запущен, или API вернул ошибку).

### Дополнительные требования (явно указанные и логически следующие):

- Использовать `tkinter` без сложных фреймворков (только стандартная библиотека Python + `requests` для вызовов API).
- Клиент не должен обращаться напрямую к БД — только через REST API.
- Приложение должно запускаться без ошибок и позволять выполнить все CRUD-операции.

## Требования к защите работы (демонстрация)

На защите студент должен:

1. **Запустить сервис (FastAPI)** – выполнить `service.py` так, чтобы API было доступно (обычно на `http://localhost:8000`).
2. **Запустить клиентское приложение** – выполнить `client.py`.
3. **Продемонстрировать работоспособность**:
   - создание новой сущности через интерфейс
   - просмотр списка сущностей
   - получение сущности по ID
   - изменение сущности по ID
   - удаление сущности по ID (проверить, что она перестаёт отображаться в активных, но не удаляется из БД)
4. Показать, что **клиент использует все точки входа API**, реализованные на оценку 4.
